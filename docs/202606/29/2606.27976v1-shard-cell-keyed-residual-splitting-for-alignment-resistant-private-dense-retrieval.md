## 摘要
稠密嵌入支撑着语义搜索和检索增强生成（RAG），然而一旦向量存储泄露，其中的大部分底层文本就会落入持有者之手。导致这种情况的攻击（少样本对齐、零样本逆向、无监督跨空间翻译）都有一个共同的弱点：受保护的存储是一个单一的全局几何结构，能够与已知结构对齐。常见的轻量级防御手段——秘密全局旋转也不例外：一旦攻击者拥有与子空间维度相当的已知对，正交普鲁克分析便能恢复该旋转。我们提出Shard，一种保持检索性能的嵌入变换，消除了这一薄弱轴。中心化嵌入被拆分为一个短公共前缀（用于第一阶段检索）和一个在C个单元中由不同秘密密钥加密的私有残差；该残差在CKKS下重排序，密钥在对消后得到精确内积。单个参数C控制着设计从所替代的全局线性基线（C=1）到按文档微密钥（C=N）的演变。由于重排序是全维度的，Shard返回原始空间的nDCG@10，这是半SVD截断所放弃的；而由于残差是按单元局部密钥加密的，若要通过分散的已知明文泄露将其映射回公共框架，代价大约是C倍的锚点数量（在C=256时中位数从200增至102,400），仅需几次加密查询。短公共前缀泄露的邻居结构信息少得多，并且微密钥极限将残差图推至零，具有不可链接、可更新的模板。该防线能够抵御有学习的、非线性和无监督的对齐器，并且在匹配效用噪声防御几乎对所有探针去匿名化时，Shard不会对任何探针去匿名化。我们坦言其局限性：在单元内部，密钥会抵消，目标攻击者仅需大约d_priv个锚点，并且重叠的参考语料库仍会通过前缀泄露。Shard是一种感知攻击的几何防御，而非密码学保证。

## Abstract
Dense embeddings underpin semantic search and RAG, yet a leaked vector store hands much of the underlying text back to whoever holds it. The attacks that make this possible (few-shot alignment, zero-shot inversion, unsupervised cross-space translation) share one weakness: the protected store is a single global geometry that can be aligned to a known one. A secret global rotation, the usual lightweight defence, is no exception: orthogonal Procrustes recovers it once the attacker has about the subspace dimension in known pairs.   We introduce Shard, a retrieval-preserving embedding transform that removes this weak axis. The centred embedding is split into a short public prefix (for stage-1 retrieval) and a private residual sharded into C cells under separate secret keys; the residual is reranked under CKKS, where the keys cancel and leave the inner product exact. A single parameter C runs the design from the global-linear baseline it replaces (C=1) to per-document micro-keys (C=N). Because the rerank is full-dimensional, Shard returns the raw-space nDCG@10 that half-SVD truncation gives up; and because the residual is keyed cell-locally, mapping it back to a common frame under a diffuse known-plaintext leak costs roughly C times more anchors (median 200 to 102,400 at C=256), for a few encrypted queries. The short public prefix leaks far less neighbour structure, and a micro-key limit drives the residual graph to zero with an unlinkable, renewable template. The barrier holds against learned, non-linear and unsupervised aligners, and where a matched-utility noise defence de-anonymises almost every probe, Shard de-anonymises none. We are plain about the limits: within a cell the keys cancel, a targeted attacker needs only about d_priv anchors, and an overlapping reference corpus still leaks through the prefix. Shard is an attack-aware geometric defence, not a cryptographic guarantee.

- **背景与动机**：
  - 全局旋转是常见的轻量级防御，但正交普鲁克分析（Orthogonal Procrustes）仅需约 `d/2` 个已知明文对即可恢复旋转矩阵，使防御失效。
  - 差分隐私（DP）噪声会破坏检索质量，且以失真代理（如σrec）作为隐私指标并不可靠；全同态加密（FHE）在密文-密文下进行top-k搜索效率过低。
  - 因此，需设计一种**攻击感知的几何防御**，从结构上消除全局对齐的弱点，同时保持检索效用并大幅提升攻击成本。

### 2. 论文提出的方法论：核心思想、关键技术细节、公式或算法流程

- **核心思想**：
  - 将居中嵌入通过PCA后拆分：**短的公共前缀`u`**（方差最大的`d_pub`维）用于本地粗检索；**私有残差`r`**（剩余`d_priv`维）按`C`个单元进行分片，每个单元用独立的秘密正交密钥`H_c`加密，存储为`z_i = H_{c(i)} r_i`。
  - 查询时，客户端将查询残差用每个活跃单元的密钥加密后发送，服务器在CKKS密态下计算内积，由于同单元密钥正交且相同，抵消后得到**精确的原始内积**，实现全维重排。
  - 参数`C`控制从全局基础（`C=1`）到每文档微密钥（`C=N`）的过渡，平衡抗对齐性、带宽与检索质量。

- **技术细节与流程**：
  1. **离线构建**：
     - 计算全局中心`μ`，PCA旋转矩阵`V`，得到`[u || r] = V(x - μ)`，其中`u`为公共前缀，`r`为私有残差。
     - 对`u`进行k-means聚类得到`C`个单元。
     - 为每个单元生成独立的Householder反射乘积正交密钥`H_c ∈ O(d_priv)`，存储`(u_i, z_i = H_{c(i)} r_i)`。
  2. **在线两阶段查询**：
     - 阶段1：客户端利用公共前缀`u`在本地近似最近邻（ANN）索引中检索，得到短列表（Kcands个候选）。
     - 阶段2：客户端对每个活跃单元发送CKKS加密的`H_c r_q`，服务器对候选分片计算密文-明文内积`⟨H_c r_q, z_i⟩ = ⟨r_q, r_i⟩`，返回加密分数。
     - 客户端解密，结合公共前缀分数，得到**全维精确相似度**`S(q,i)=⟨x_q-μ, x_i-μ⟩`。
  3. **关键公式/命题**：
     - **命题2（精确单元密钥重排）**：正交密钥保证了`⟨H_{c(i)} r_q, z_i⟩ = ⟨r_q, r_i⟩`，从而重构无截断的全维分数。
     - **命题3（分散锚点复杂度）**：若攻击者锚点均匀分散在`C`个单元中，恢复单个单元密钥需要`≥ d_priv`个单元内锚点，总锚点需求为`Θ(C·d_priv)`，即约`C`倍于全局密钥。**针对性攻击者**若集中锚点于一个单元，则成本仅为`d_priv`。
  4. **CKKS加速**：采用密文-明文（ct-pt）操作，避免重线性化，仅需一个乘法级别，参数自动选择使得速度约为默认配置的1.7倍。

### 6. 论文的主要结论与发现

- **消除薄弱轴**：Shard通过单元密钥残差分裂，成功将扩散已知明文攻击的锚点需求从`~200`提升至`~102,400`（C=256时），实现了约`C`倍的对抗增益。
- **恢复原始效用**：全维重排使Shard在BEIR任务上匹配原始nDCG@10，弥补了传统半SVD截断带来的2-8点损失，且公共前缀仅宽1/4至1/2。
- **对抗学习型攻击**：学习线性、非线性MLP和无监督对齐方法均未能打破`d_priv`的单单元锚点阈值，正交Procrustes反而最强。
- **公共泄露更少**：短前缀泄露的精细邻域结构远少于基础公开的半空间，微密钥极限可消除残差泄露并实现可撤销、不可链接的模板。
- **显著优于失真防御**：在同等检索效用下，DP噪声防御可被轻松去匿名化（R@1≈1），而Shard达到零去匿名化。
- **诚实局限**：单元内密钥抵消导致同单元相似性可算；针对性攻击者成本仅为`d_priv`；拥有重叠参考语料的攻击者仍能通过公共前缀去匿名。

### 7. 优点：方法或实验设计上有哪些亮点

- **攻击感知设计哲学**：根本上针对对齐攻击面（消除全局几何）设计，而非依赖不可靠的失真代理`σrec`。
- **优雅的参数化**：单一参数`C`在全局防护与每文档微密钥之间连续调节，提供清晰的带宽-安全性权衡。
- **保持精确检索**：利用正交密钥抵消与CKKS同态加密，实现了首个做到密码学查询隐私、精确全维重排和抗对齐文档保护的混合方案。
- **全面的实验验证**：
  - 包含**严格的统计检验**，给出了置信区间，结论可靠。
  - 覆盖**多样化的编码器、语言、任务和攻击者**，验证了方法的泛化性。
  - 明确区分了扩散攻击与针对性攻击的场景，设定了诚实的预期。
  - 与多种基线（DP噪声、PQ-only、全局MLP）进行了公平比较。
- **可实践性**：无需重新训练编码器，基于标准线性代数和CKKS实现，成本清晰可测，代码开源支持复现。

### 8. 不足与局限：包括实验覆盖、偏差风险、应用限制等

- **非密码学保证**：本质是攻击感知的几何防御，无法提供形式化安全证明，单元内密钥抵消等问题是结构性泄露。
- **单元内与粗图泄露**：同一单元内，密钥抵消导致计算出的相似度泄露精细邻域信息。只有微密钥极限才能消除，但会带来带宽成本。
- **针对性攻击者成本低**：若攻击者能精确定位至受害者的公共单元并集中锚点，成本仅为`d_priv`（约320或576），并非`C`倍。这限制了在高度定向攻击下的防护。
- **重叠参考泄露未闭合**：当一个重叠的公共语料库存在时，攻击者只需破解单个公共前缀密钥（成本`d_pub`）即可去匿名化，这与残差保护无关。
- **实验覆盖限制**：
  - 主要评估的是嵌入空间的重新识别和对齐，而非端到端的**文本生成反演**（如微调反演器），后者可能构成更强威胁。
  - 自检索探针具有局限性，且100万规模的BEIR式实验因算力限制而未完成。
  - 在线延迟测量基于环回网络，未包含广域网延迟，且评估仅涉及少量CKKS查询，扩展到更大短列表时的成本需进一步权衡。
- **应用限制**：依赖客户端持有秘密并执行本地阶段1搜索；需要与PIR/ORAM结合才能隐藏访问模式；某些压缩型或复述型编码器（如e5-small, mpnet）在激进截断下效用损失大，可能需要替换编码器。
